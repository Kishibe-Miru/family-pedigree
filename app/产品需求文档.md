# 精神科遗传家族谱系图绘制工具 2.0 产品需求文档

## 1. 项目定位

本项目是一个本地使用的网站工具，用于绘制精神科遗传家族谱系图。

第一阶段目标不是进行遗传分析、诊断推理或风险预测，而是帮助用户更简单、规范、稳定地绘制家族谱系图，并导出为图片。

产品核心定位：

> 面向精神科临床、教学、科研记录场景的本地家系图绘制器。

## 2. 使用场景

用户可能在以下场景中使用本工具：

- 病例整理时绘制精神疾病家族史。
- 临床汇报、教学课件中展示家系关系。
- 科研资料收集时快速制作家族谱系图。
- 将家系图导出为图片后插入 Word、PPT 或其他文档。

工具应优先保证绘制过程简单、图形清晰、导出稳定。

## 3. 目标用户

主要用户：

- 精神科医生
- 心理卫生相关临床人员
- 医学研究人员
- 需要整理精神疾病家族史的学生或助理

用户不一定熟悉专业绘图软件，因此操作方式应尽量结构化、按钮化，减少手动画线。

## 4. 产品目标

### 4.1 第一阶段目标

- 支持从零创建一张家族谱系图。
- 支持添加家庭成员及核心亲属关系。
- 支持精神科家系图常用符号。
- 支持自动排列谱系图。
- 支持必要的手动微调。
- 支持导出 PNG 图片。
- 支持保存和重新加载本地项目文件。

### 4.2 暂不处理的目标

第一阶段暂不做：

- 遗传风险自动分析。
- 精神疾病诊断建议。
- 云端保存。
- 多用户协作。
- 账号系统。
- 权限管理。
- 复杂数据库管理。
- 自动生成医学结论。

## 5. 核心设计原则

### 5.1 结构化绘制优先

用户不应主要依赖手动画线完成家系图。

推荐交互方式是：

1. 先添加一个成员。
2. 选中该成员。
3. 通过按钮添加父亲、母亲、配偶、子女、兄弟姐妹。
4. 系统自动生成节点和关系线。

这样可以降低错误率，并让关系数据保持清晰。

### 5.2 自动排版为主，手动调整为辅

系统应根据代际关系自动排列成员。

用户可以在自动排版后拖动节点进行微调，但手动位置不应破坏底层亲属关系数据。

### 5.3 导出图片是核心结果

第一阶段的最终产物是清晰图片，而不是复杂数据分析。

导出图片应适合插入：

- Word
- PowerPoint
- 病例讨论材料
- 科研记录

### 5.4 本地优先

工具应尽量支持本地浏览器直接运行。

用户数据默认保存在本地，不上传云端。

## 6. 核心工作流

### 6.1 新建谱系图

用户打开网页后，可以选择：

- 新建空白家系图。
- 打开之前保存的 JSON 项目文件。

新建时默认创建一个空画布。

### 6.2 创建先证者

用户首先添加先证者。

先证者字段包括：

- 编号或姓名
- 性别
- 年龄或出生年份
- 是否患病
- 诊断标签
- 是否死亡
- 备注

先证者在图中应有箭头标记。

### 6.3 添加亲属关系

选中某个成员后，用户可以快速添加：

- 父亲
- 母亲
- 配偶
- 子女
- 兄弟姐妹

添加后系统自动创建对应节点和关系线。

### 6.4 编辑成员信息

用户选中节点后，在右侧属性面板编辑成员信息。

可编辑字段包括：

- 显示名称
- 性别
- 年龄
- 出生年份
- 是否死亡
- 是否患病
- 疾病状态
- 精神科诊断
- 是否先证者
- 备注

### 6.5 调整图形

用户可以：

- 缩放画布。
- 平移画布。
- 拖动单个节点。
- 一键重新自动排版。
- 重置视图。

### 6.6 导出图片

用户点击导出按钮后，可导出当前谱系图为 PNG 图片。

后续可考虑支持 SVG、PDF，但第一阶段以 PNG 为主。

## 7. 页面布局

建议采用三栏结构。

### 7.1 左侧操作栏

功能：

- 新建图谱
- 保存项目
- 加载项目
- 添加成员
- 添加父亲
- 添加母亲
- 添加配偶
- 添加子女
- 添加兄弟姐妹
- 自动排版

### 7.2 中间画布区

功能：

- 展示谱系图。
- 支持节点选择。
- 支持节点拖动。
- 支持画布缩放和平移。
- 显示关系线和代际结构。

### 7.3 右侧属性栏

功能：

- 显示当前选中成员的信息。
- 编辑成员属性。
- 设置诊断和疾病状态。
- 删除成员。
- 设置为先证者。

## 8. 家系图符号规范

第一阶段支持以下基础符号。

### 8.1 性别符号

- 男性：方形
- 女性：圆形
- 未知性别：菱形

### 8.2 疾病状态

- 未患病：空心符号
- 患病：实心或明显填充
- 疑似患病：半填充或斜线填充
- 未知：灰色或问号标记

### 8.3 其他状态

- 先证者：箭头指向该成员
- 死亡：节点上添加斜杠
- 流产或死胎：小三角或特殊符号
- 收养：可后续扩展为虚线关系
- 离异：配偶关系线上添加斜杠
- 双胞胎：可后续扩展为特殊分叉线

第一阶段可以先完整实现性别、患病、疑似、未知、死亡、先证者。

## 9. 精神科诊断标签

第一阶段提供常见精神科诊断标签。

初始标签建议包括：

- 精神分裂症
- 双相情感障碍
- 抑郁障碍
- 焦虑障碍
- 强迫障碍
- 自闭谱系障碍
- 注意缺陷多动障碍
- 物质使用障碍
- 人格障碍
- 其他

诊断标签主要用于显示和备注，不参与自动分析。

## 10. 数据模型

底层应保存结构化数据，而不是只保存画布图形。

### 10.1 人员数据

每个成员可包含：

```json
{
  "id": "p1",
  "name": "II-3",
  "sex": "female",
  "age": "",
  "birthYear": "",
  "affectedStatus": "affected",
  "diagnoses": ["双相情感障碍"],
  "deceased": false,
  "proband": true,
  "notes": "",
  "x": 0,
  "y": 0
}
```

### 10.2 关系数据

关系应单独保存。

```json
{
  "id": "r1",
  "type": "partner",
  "person1": "p1",
  "person2": "p2"
}
```

关系类型建议包括：

- parentChild
- partner
- separated
- divorced

### 10.3 项目数据

完整项目文件结构：

```json
{
  "version": "2.0",
  "title": "未命名家系图",
  "people": [],
  "relationships": [],
  "settings": {
    "showNames": true,
    "showDiagnoses": true
  }
}
```

## 13. 技术建议

### 13.1 技术形态

第一阶段建议使用纯前端本地网页：

- `index.html`
- `styles.css`
- `app.js`

优点：

- 不需要服务器。
- 双击 HTML 或通过本地静态服务即可运行。
- 数据不离开本机。
- 后续容易打包成桌面应用。

### 13.2 渲染方式

建议优先使用 SVG 绘制谱系图。

原因：

- 关系线和节点符号适合 SVG。
- 缩放后清晰。
- 后续可导出 SVG。
- 转 PNG 也相对方便。

### 13.3 导出方式

第一阶段导出 PNG：

1. 将 SVG 转为图片。
2. 绘制到 Canvas。
3. 下载 Canvas 生成的 PNG。

## 14. 关键交互细节

### 14.1 添加关系时的行为

当用户选中一个人后：

- 点击“添加父亲”：在上一代创建男性节点，并建立亲子关系。
- 点击“添加母亲”：在上一代创建女性节点，并建立亲子关系。
- 点击“添加配偶”：在同一代创建节点，并建立配偶关系。
- 点击“添加子女”：在下一代创建节点，并建立亲子关系。
- 点击“添加兄弟姐妹”：创建同代节点，并自动共享父母关系；如果父母不存在，可提示先创建父母或自动创建未知父母。

### 14.2 删除成员时的行为

删除成员时应同时删除与该成员相关的关系。

如删除后出现孤立成员，可以保留，不自动删除。

### 14.3 自动排版

自动排版应遵循：

- 上一代在上方。
- 下一代在下方。
- 配偶在同一水平线。
- 兄弟姐妹尽量水平排列。
- 先证者尽量位于视觉中心。

第一阶段可以接受简单规则布局，后续再优化复杂家系。

## 15. 验收标准

第一阶段完成后，应满足：

- 用户可以从空白页面创建至少三代家系图。
- 图中可以正确显示男性、女性、未知性别。
- 图中可以显示患病、疑似、未知、未患病状态。
- 可以标记先证者。
- 可以标记死亡。
- 可以添加父母、配偶、子女、兄弟姐妹。
- 可以保存并重新加载项目。
- 可以导出 PNG 图片。
- 导出的图片清晰，不出现明显裁切。
- 页面在常见桌面浏览器中可用。

## 16. 第二版开发建议顺序

建议按以下顺序开发：

1. 建立基础文件结构。
2. 实现数据模型。
3. 实现成员增删改。
4. 实现关系添加逻辑。
5. 实现 SVG 渲染。
6. 实现基础自动排版。
7. 实现节点选择和右侧属性编辑。
8. 实现拖拽和缩放。
9. 实现保存、加载 JSON。
10. 实现 PNG 导出。

## 17. 当前阶段结论

第二版应从一个可用的本地 MVP 开始，不追求一次性覆盖全部复杂家系情况。

最重要的是先完成：

- 结构化录入
- 自动生成谱系图
- 基础手动调整
- 图片导出

这四件事完成后，本工具就已经具备实际使用价值。
