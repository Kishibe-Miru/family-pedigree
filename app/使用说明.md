# 家族谱系图绘制工具 4.6 · 使用说明

面向精神科临床 / 教学 / 科研的本地家系图绘制器。纯前端、不联网、数据只存本机。

## 本版（4.6，2026-06-23）家庭块级避让

1. **核心家庭块不再被同胞插入**：4.5 的最后重叠修复按“单个人”右推，可能把较小同胞塞进“先证者 + 配偶”的夫妻节点内部，使婚配节点中心落在同胞竖轴上，第三代子女看起来像挂到同胞下面。4.6 改为按夫妻节点 / 家庭块整体避让。
2. **保留夫妻节点内部相对位置**：同代避让移动的是整块节点及其后代，不再单独移动配偶、同胞或子女，从而保持“本人-配偶-子女”的局部紧凑。
3. **同胞横线遇到姻亲竖线自动留缺口**：当配偶原生家庭的下降线穿过先证者同胞线时，同胞线会在交叉点附近分段，避免视觉上把两组关系连成一条线。
4. **需要整体避让的家庭块**：本人核心家庭块、配偶原生家庭块、同胞各自成家块、半同胞家庭块、再婚 / 多配偶家庭块、姻亲扩展块、单亲子女块、堂表 / 近亲婚配块。

## 本版（4.5，2026-06-23）父母下降线对齐血缘子女

1. **修复“父母到子女折线”问题**：当子女已经有配偶时，4.4 会把父母婚配中心对齐到“子女+配偶”的夫妻超级节点中心，导致下降线先垂直落到夫妻中心，再水平折到血缘子女本人。4.5 改为对齐血缘子女本人的锚点。
2. **父母-子女主线更符合常规家系图**：单个血缘子女时，父母婚配线中心向下应直接垂直到该子女符号上方；子女配偶仍通过子女本人的婚配线连接，不参与上一代父母下降线的居中计算。
3. **多子女家庭仍按血缘子女组居中**：若同胞中有人已成家，父母下降线会对齐所有血缘子女本人的重心，而不是这些子女各自小家庭的重心。

## 本版（4.4，2026-06-22）婚配节点与大家庭同胞线优化

1. **婚配关系改用不可见 `marriage node`**：每段配偶关系都有稳定的 `marriageNodeId`，绘制时生成运行时婚配节点坐标；子女下降线、近亲双线、手动错位台阶线都从同一个节点派生，不再各自临时计算父母中点。
2. **多配偶婚配节点自动错层**：同一成员存在多个配偶，或直连婚配线会穿过其他成员时，婚配节点自动抬到上方独立层，子女下降线从该层向下连接，减少“多位配偶被一条直线串起来”的误读。
3. **同胞横线分段绘制**：大家庭中 6 个及以上同胞时，同胞主线按相邻锚点分段输出，保留血缘同胞的一条主线语义，同时便于后续按组做更细避让。
4. **大同胞组增加轻微组间距**：超过阈值的同胞按出生顺序每 3 人形成视觉小组；兄弟姐妹各自成家时，会连同配偶和子女小家庭一起移动，优先保持“本人-配偶-子女”的局部紧凑。

## 本版（4.3，2026-06-22）家庭单元模型显式化

1. **新增运行时 `familyUnit` 模型**：从 `parentChild` 与 `partner` 关系同步出稳定家庭单元，包含 `id`、`parentKey`、`parentIds`、`childIds`。布局与连线不再各自临时猜测父母组合，而是统一读取 `state.familyUnits`。
2. **父母组合成为一等结构**：父母 A+B 是一个家庭单元，子女挂到该家庭单元下；单亲家庭也作为单亲家庭单元处理。
3. **多配偶 / 半同胞自然分组**：同一成员与不同配偶形成不同家庭单元；半同胞按不同父母组合分组，后续连线避让和排版优化有稳定基础。
4. **JSON 兼容升级**：旧项目仍按 `people + relationships` 加载；保存时会附带派生的 `familyUnits`，便于调试和后续版本进一步迁移。

## 本版（4.2，2026-06-22）连线与大家庭排版修复

1. **兄弟姐妹按录入 / 出生顺序从左到右**：修复 4.1 中同胞会被配偶、子女重心反向影响而重新排序的问题。现在同一父母组合下的子女横向顺序优先保持录入顺序，适配“父母 + 多个同胞 + 各自成家”的大家庭场景。
2. **移动同胞时同步移动其后代小家庭**：同胞顺序修正不再只移动本人或夫妻节点，而是连同其子女小家庭一起平移，避免出现长距离亲子横线。
3. **多配偶 / 半同胞关系避让**：当一段婚配线直连会穿过其他成员符号时，自动改画为上方独立折线，并让对应子女下降线从该折线的婚配中心下降，减少“父亲、前妻、后妻被一条线串起来”的误读。
4. **新增回归测试场景**：覆盖三代大家庭、叔伯姑姨 / 堂表亲、兄弟姐妹顺序、半同胞、四代直系、多兄弟姐妹各自成家、孪生、近亲婚配等场景。

### 仍需后续精修

- 近亲婚配中的堂表亲布局仍可能出现较长横线，但比 4.1 更稳定。
- 自由拖动后配偶不同高时仍使用台阶折线，后续可进一步定义更规范的手动布局连接点。

## 本版（4.1，2026-06-12）布局修复

1. **T 形下降线**：修复了先证者有多个同胞时，下降竖线落在同胞横线最右端而非中点（L 形）的问题。根因：步骤4上行排序用叶节点自身 cx 作 key，导致兄弟节点排到夫妻节点右侧；步骤5单向扫描将首个子女固定在父母位置，使子女重心偏右 80px。修复：上行排序改用父代重心作叶节点的回退 key；步骤5后增加一次显式居中，将子女整组平移至以父节点为中心。
2. **配偶不再被同胞横线穿越**：同胞数 ≥ 3 时，配偶曾落在同胞横线范围内（视觉上横线穿过配偶符号）。随 T 形修复一并解决——配偶现始终在先证者右侧，超出同胞横线范围。
3. **独生子女无断线**：单亲子女与父母中点的水平偏差降为 0，消除了近亲婚配等场景下的孤立连线段。

## 本版（4.1）功能修复与新增

1. **新增「与已选成员结婚」（连接已有节点为配偶）**：此前「配偶」按钮只能新建一个配偶节点，无法把两个已经画在图上的成员连为夫妻，导致**近亲婚配（如表兄妹结婚）等场景根本无法构建**，4.0 的近亲双线功能实际无法触发。现在：选中一位成员 → 点「💍 与已选成员结婚」（按钮高亮进入连接模式）→ 在画布中点击另一位成员，两人即连为配偶；若检测到共同祖先（近亲），自动画双线并在状态栏提示。已内置校验：拒绝与自己、与直系亲属（父母/子女）结婚；再次点击按钮或点击空白处可取消。
2. **家族史文字不再把占位名当真名**：自动生成的父母/同胞默认名为「父」「母」「兄弟」等占位符。此前家族史会输出「父（祖父）」「父（父亲）」这类别扭文本。现在占位名一律视为「无名」，只输出称谓（如「父亲患…」）；若填写了真实姓名则显示「真名（称谓）」（如「李建国（祖父）患…」）。
3. **删除孪生后清理孤立 twinGroup**：删除一对孪生中的一个后，剩余成员不再残留「双胞胎」标记，避免出现「无人配对的双胞胎」脏数据。
4. **近亲检测边界**：`areConsanguineous(x, x)` 不再把同一人误判为近亲。

## 继承自 4.0 的功能

详见 4.0 使用说明。新增功能包括：双胞胎、携带者符号、近亲婚配双线、拖动自由摆放、一键生成家族史文字。

## 原 4.0 新增功能（完整列表）

1. **双胞胎**：左栏新增「孪生兄弟 / 孪生姐妹」按钮，与选中成员共享父母，从同胞横线的同一顶点斜线分叉绘制。选中孪生成员后，右侧属性栏出现「同卵双胞胎」勾选项——勾选后两条分叉线之间自动加一道水平横杆（同卵记号），不勾选即为异卵。
2. **携带者符号**：疾病状态新增「携带者」，符号中心绘制一个实心圆点（NSGC 标准携带者记号）。
3. **近亲婚配双线**：当一对配偶存在共同祖先（4 代内）时，配偶线自动绘制为双线。
4. **拖动自由摆放**：左栏「拖动后自动对齐」开关。开启时（默认）保持 3.x 行为——拖动后吸附回标准布局；关闭后，被拖动的节点保留手动位置，自动排版时不再被覆盖，连线自动跟随，便于手工微调复杂家系。
5. **一键生成家族史文字**：左栏「生成家族史文字」按钮，自动按**亲缘级别**（用亲缘系数 r 计算：一级 0.5、二级 0.25、三级 0.125，因此全同胞正确归为一级）归纳所有患病/疑似亲属，识别父系/母系及祖父母、孙辈、叔伯姑姨等称谓，生成可直接粘贴进病历的描述段落，并附家族聚集性提示。弹窗内可一键复制到剪贴板。
6. 图例同步新增携带者、双胞胎、近亲婚配三个条目。

## 一、如何运行

**方式一：直接打开**
双击 `index.html`，用 Chrome / Edge / Firefox 打开即可。

**方式二：局域网部署（推荐多人共用）**
把整个 `2026家族谱系图4.0 260612` 文件夹放到 Apache / Nginx / IIS 的站点目录，局域网用户访问：

```
http://你的局域网IP/family-pedigree/
```

文件夹内只有 4 个核心文件即可运行：`index.html`、`styles.css`、`app.js`、（外加规则与说明文档，可选）。

## 二、基本流程（从先证者开始）

1. 点击左上「**＋ 添加先证者**」，画布出现第一个节点（自动带先证者箭头与 P 标记）。
2. 在画布中**点击选中**任一成员，左侧「为选中成员添加亲属」按钮被激活。
3. 用按钮或快捷键扩展家系（**男 / 女按钮已拆开，添加时无需再手动设性别**）：
   - **父亲 / 母亲** —— 在上一代生成，并自动与已有的另一位父母连成配偶。
   - **配偶** —— 同代生成，自动判断为相反性别。
   - **儿子 / 女儿** —— 下一代生成，自动连到夫妻中点。
   - **兄弟 / 姐妹** —— 同代生成，自动共享父母；若当前成员没有父母，会自动补一对未知父母。添加后光标**保持在原成员**，方便连续添加多个同胞。
4. 在**右侧属性栏**编辑姓名、性别、疾病状态、年龄、诊断、是否已故 / 先证者、备注。
5. 每次添加都会**自动排版**；也可随时点「↳ 自动排版」一键整理。
6. 点「**导出 PNG**」或「**导出 SVG**」保存图片，插入 Word / PPT / 病例材料。

## 三、键盘快捷键（录入更快）

| 键 | 作用 | 键 | 作用 |
|---|---|---|---|
| `F` | 加父亲 | `C` | 加儿子 |
| `M` | 加母亲 | `D` | 加女儿 |
| `E` | 加配偶 | `B` | 加兄弟 |
| `Ctrl+Z` | 撤销 | `S` | 加姐妹 |
| `Ctrl+Y` / `Ctrl+Shift+Z` | 重做 | `Del` | 删除选中成员 |

> 在输入框内打字时快捷键不会触发，可放心输入姓名/备注。

## 四、画布操作

- **选中**：单击节点。
- **移动单个节点**：拖动节点；松手后会按同代左右顺序重新吸附为标准布局。
- **平移画布**：在空白处按住拖动。
- **缩放**：鼠标滚轮，或左侧 `－ ＋`；点「适应」自动缩放并居中。

## 五、符号规范（NSGC / Bennett 标准）

- 男性 = 方形 □，女性 = 圆形 ○，性别未知 = 菱形 ◇。
- 未患病 = 空心，患病 = 实心，疑似 = 斜线填充，状态未知 = 灰底。
- 已故 = 符号上一条斜杠 ／。
- 先证者 = 左下箭头 + `P` 标记（一图默认一位）。
- 配偶线水平连接；子女线从夫妻线中点垂直下降到同胞横线，再分别下连到每个子女。
- 左侧罗马数字标代际（I、II、III…），节点上方可显示个体编号（如 II-3）。

显示选项（左侧「显示选项」）可分别开关：世代编号、名称、诊断标签、年龄/出生年、图例、标题区。

## 六、保存与读取

- **自动保存**：每次改动写入浏览器 localStorage，下次打开自动恢复。
- **保存项目**：导出 `.json` 结构化文件（含所有人员、关系、设置），便于归档与下次继续编辑。
- **加载项目**：选择之前保存的 `.json` 即可还原。
- **导出图片**：PNG（2.5 倍高清、白底，适合插入文档）与 SVG（矢量，可在 Illustrator/AI 二次编辑、无损缩放）。

## 七、本版（3.0）相对 2.0 的改进

1. **自动排版更可靠**：用基于「家庭单元」的递归 tidy-tree 算法替换原迭代推挤，多代多家庭对齐更稳，连线更少交叉、节点不重叠。
2. **图形更规范美观**：统一比例、规范先证者箭头与 P 标记、世代/个体编号、可选图例与标题区，导出更适合发表与教学。
3. **录入更顺手**：键盘快捷键、撤销/重做、空白处平移、滚轮以光标为中心缩放、自动补全未知父母、防止重复添加父母等。
4. **导出**：新增 SVG 矢量导出；PNG 按内容边界裁剪、白底高清。

## 八、暂未包含（按需可扩展）

维持现有符号集，未包含：双胞胎、近亲婚配双线、流产/死胎/妊娠、收养、携带者圆点、再婚多配偶的高级避让、大型家系分页、自动家族史文字、遗传风险统计等。如需要可在此版基础上扩展。
